How to use my website's template

I feel like common academic website templates, such as Academic pages, are clumsy, with too many functionalities. I wanted something much more simple to edit and use. Hence, I created this template, which you can freely fork from GitHub and use.

Forking the repository

First thing first, fork the template's GitHub repository. Once it is done, you can edit the different files to add your own information and style. The website structure looks like that:

The index.html is the homepage of the website. It is important that its name remains the same to setup the website with GitHub Pages. The css directory contains the style of the website.

Customization

This template's goal was to make it easy to customize. If you want to add a page to your website, add a file with your new page name in the pages directory. Then, add a link to this page in the navigation bar (in the utils/topnav.html file). To remove a page, remove the page from the pages directory and from the utils/topnav.html file.

The easiest way to add a page would be to copy/paste and rename the posts/template.html file, as it only contains the main structure of the page. Otherwise, this is just a HTML/CSS website, you can easily edit it to match your own tastes !

Note: while editing your website, if you have python installed, you can start running a local server to see you modifications in real-time. Just go to your local repository and use the following command: python -m http.server. Then, copy/paste the provided link in any navigator.

Setting-up the website

Once you finished customizing your website's template, push your changes on your GitHub. From there, go to your GitHub repository, navigate to Settings/Pages and make sure that the Source is Deploy from a branch. You can specify which branch to use under the Branch parameter.

Et voilà ! You now have your very own website. Pushing new changes to the correct branch will initiate an update of your website, without the need to do anything. To take a look at the current state of your website, you can visit the Deployments tab of your repository. Warning: each update takes some minutes to perform.
